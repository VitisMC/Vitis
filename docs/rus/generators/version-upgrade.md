# Руководство по обновлению версии

Пошаговая инструкция по обновлению Vitis до новой версии Minecraft.

## Предварительные требования

- Java 21+ (для генератора данных серверного JAR и MinecraftDecompiler)
- Go 1.21+
- curl, python3

## Простой способ

```bash
./scripts/update_version.sh <НОВАЯ_ВЕРСИЯ>
```

Эта единственная команда делает всё: скачивает данные, генерирует `registries.json` из серверного JAR, скачивает датапаки и теги из misode/mcmeta, декомпилирует серверный JAR, извлекает таблицы лута и запускает все 59 генераторов.

Передайте целевую версию Minecraft как аргумент (напр. `1.22`).

## Что делает `update_version.sh`

| Шаг | Описание | Источник |
|-----|----------|---------|
| 1 | Скачивание JSON-файлов PrismarineJS | `PrismarineJS/minecraft-data` |
| 2 | Генерация `registries.json` | Генератор данных серверного JAR |
| 3 | Скачивание датапаков конфиг-регистров | `misode/mcmeta` тег `{версия}-data-json` |
| 4 | Скачивание тегов | `misode/mcmeta` тег `{версия}-data-json` |
| 5 | Декомпиляция серверного JAR | MinecraftDecompiler + Vineflower |
| 6 | Извлечение таблиц лута и рецептов | Внутренний архив серверного JAR |
| 7 | Запуск Go-парсеров декомпилированного кода | `scripts/parsers/*.go` |
| 8 | Запуск всех 59 генераторов | `internal/data/generator/*/main.go` |
| 9 | Проверка сборки | `go build ./...` |

## После запуска скрипта

### Проверка новых регистров или датапаков

Если новая версия добавляет конфигурационные регистры или типы worldgen, обновите массивы в `update_version.sh`:

- `REGISTRIES` — датапаки для скачивания из mcmeta (Шаг 3)
- `TAG_REGISTRIES` — директории тегов для скачивания (Шаг 4)
- `GENERATORS` — кодогенераторы для запуска (Шаг 8)

### Обновление версионных констант

- **Версия протокола** — проверьте [Minecraft Wiki: версии протокола](https://minecraft.wiki/w/Java_Edition_protocol_version)
- **Версия Known Packs** — в `internal/session/configuration_handler.go` обновите `Version: "1.21.4"` на новую версию
- **Registry builtin.go** — если добавились новые регистры, добавьте записи в `builtinIDMap()` в `internal/registry/builtin.go`

### Ручная проверка

1. Запустите `go test ./...`
2. Запустите сервер и подключитесь ванильным клиентом
3. Проверьте логи на ошибки регистров/тегов
4. Проверьте новые/удалённые типы пакетов и поля

## Устранение неполадок

### Генерация `registries.json` не удалась

Требуется Java 21+. Проверьте, что серверный JAR скачался:
```bash
ls -la .mc-decompiled/downloads/<версия>/server.jar
```

Если отсутствует — перезапустите `update_version.sh`, он скачивает JAR на Шаге 6 (extract_loot_tables.sh).

### Генератор показывает ✗

Запустите генератор вручную, чтобы увидеть ошибку:
```bash
go run internal/data/generator/<имя>/main.go -version <версия>
```

Частые причины:
- `registries.json` отсутствует → Шаг 2 не удался
- JSON датапака отсутствует → проверьте, что массив `REGISTRIES` содержит нужную запись
- Декомпилированный код отсутствует → Шаг 5 не удался (проверьте, установлена ли Java)

### Клиент вылетает с ошибками регистров

- Отсутствуют данные тегов → убедитесь, что массив `TAG_REGISTRIES` полон
- Неверные типы NBT → проверьте `doubleFields`/`longFields` в `internal/registry/generator/main.go`
- Отсутствует конфиг-регистр → проверьте map `configRegistries` в генераторе регистров

### Скачивание датапаков показывает 0 файлов

GitHub API может быть ограничен по количеству запросов. Подождите или используйте `GITHUB_TOKEN`:
```bash
export GITHUB_TOKEN=ghp_...
```
